HTML / CSS — cần nắm vững
  • Flexbox & Grid layout system
  • CSS custom properties (variables)
  • Responsive design (media queries, viewport)
  • CSS transforms (translate, rotate, scale) — tiền thân của 3D transform
  • CSS animation & transition (easing, keyframes)
JavaScript — những gì Three.js cần
  • requestAnimationFrame — vòng lặp render 60fps
  • Event listeners (mouse, scroll, touch, resize)
  • ES6 Modules (import/export) — cách Three.js được tổ chức
  • Destructuring, spread, optional chaining
  • Promise / async-await (load model bất đồng bộ)
  • TypedArray (Float32Array) — dữ liệu geometry thô
React — nếu muốn dùng React Three Fiber
  • useState, useEffect, useRef (dùng rất nhiều trong R3F)
  • useCallback, useMemo — tránh re-render trong scene 3D
  • Context API — share scene state toàn cục
  • Vite hoặc Next.js setup cho 3D project
Toán — không cần giỏi, cần hiểu trực quan
  • Vector 3D (x,y,z) — vị trí, hướng, pháp tuyến
  • Dot product — tính góc giữa 2 vector (dùng trong lighting)
  • Cross product — tính vector vuông góc (tính normal face)
  • Matrix 4x4 — transform position/rotation/scale của object
  • Euler angles vs Quaternion — cách mô tả xoay
  • UV coordinates — ánh xạ texture lên bề mặt 3D
Tip: Học toán 3D song song với Three.js, đừng cố học toán trước — bạn sẽ thấy ngay tại sao vector/matrix cần thiết khi code.

Three.js — kiến trúc cần nắm
  • Scene graph: Scene → Group → Mesh → Geometry + Material
  • Camera types: PerspectiveCamera vs OrthographicCamera
  • WebGLRenderer: antialias, pixel ratio, shadow maps
  • Lights: AmbientLight, DirectionalLight, PointLight, SpotLight
  • Loaders: GLTFLoader (load .glb), TextureLoader, DRACOLoader
  • Raycasting — detect click/hover lên object 3D
  • OrbitControls — camera rotate/zoom/pan
  • Dispose pattern — giải phóng memory đúng cách
GSAP — công thức để làm Sutera/Utopia Tokyo
  • gsap.to() / gsap.from() / gsap.fromTo() — tween cơ bản
  • Timeline — chain nhiều animation theo thứ tự
  • ScrollTrigger: trigger, scrub, pin — animation theo scroll
  • Tween thuộc tính của Three.js object: position.x, rotation.y, material.opacity
  • scrub: true — animation scrub theo scroll position (mượt như Utopia Tokyo)
  • Kết hợp với Lenis: gsap.ticker.add(() => lenis.raf())
Blender — chỉ cần đủ để xuất model cho web
  • Navigation cơ bản: orbit, pan, zoom trong viewport
  • Modelling cơ bản: extrude, loop cut, subdivision
  • UV Unwrapping — trải mặt 3D thành 2D để texture
  • Bake texture: tính ánh sáng/shadow sẵn, lưu thành ảnh PNG
  • Export .glb với DRACO compression (giảm 80–90% file size)
  • Poly count: giữ dưới 50k triangles cho web mobile
Spline — đường tắt cho Tầng 1 nhanh
  • Thiết kế scene 3D trực tiếp trong browser (no-code/low-code)
  • Export dưới dạng React component hoặc script embed
  • Có physics, interactions, animation built-in
  • Phù hợp: portfolio cá nhân, landing page, prototype
  • Hạn chế: ít control hơn Three.js, bundle size lớn (~500KB+)
React Three Fiber — Three.js dưới dạng JSX
  • @react-three/fiber: Canvas, useFrame, useThree
  • @react-three/drei: helpers tiện ích (OrbitControls, Text, Html, useGLTF)
  • useFrame — chạy code mỗi frame render (animation loop)
  • useThree — access scene, camera, renderer trong component
  • Suspend + lazy load model với Suspense
  • @react-three/postprocessing — bloom, depth of field, vignette
Lenis — smooth scroll nền cho toàn trang
  • Intercept native scroll, tạo momentum (inertia)
  • Kết hợp GSAP ticker: gsap.ticker.add((time) => lenis.raf(time * 1000))
  • gsap.ticker.lagSmoothing(0) — tránh lag khi tab background
  • Cấu hình: lerp (0.1 = rất mượt), duration, easing

GLSL — ngôn ngữ chạy trên GPU
  • Vertex shader: nhận vị trí từng điểm, transform theo matrix
  • Fragment shader: quyết định màu từng pixel — đây là nơi magic xảy ra
  • Uniforms: biến truyền từ JS vào shader (time, mouse, resolution)
  • Varyings: truyền dữ liệu từ vertex sang fragment shader
  • Built-in functions: sin, cos, mix, smoothstep, clamp, fract, mod
  • Noise functions: Perlin noise, Simplex noise — tạo texture organic
  • SDF (Signed Distance Field) — vẽ hình học bằng toán
  • Học từ: The Book of Shaders (thebookofshaders.com) — free, tuyệt vời
Particle System — hiệu ứng đặc trưng của Creative Web
  • BufferGeometry + Float32Array — quản lý hàng nghìn điểm hiệu quả
  • THREE.Points + PointsMaterial — render particle cơ bản
  • Custom shader particle — control màu, kích thước từng điểm
  • GPU Compute (GPGPU) — simulate hàng triệu particle bằng shader
  • Morphing giữa 2 particle shape (sphere → text → face)
  • Frustum culling tự động — Three.js tự ẩn particle ngoài camera
Post-processing — lớp filter sau khi render
  • EffectComposer — pipeline xử lý ảnh sau render
  • Bloom — ánh sáng tràn ra ngoài vật thể sáng
  • Chromatic Aberration — tách màu RGB (hiệu ứng lens glitch)
  • Depth of Field — blur theo khoảng cách từ camera
  • Film grain — texture hạt film analog
  • SSAO (Screen Space Ambient Occlusion) — shadow mềm tiếp xúc
  • @react-three/postprocessing — wrapper tiện lợi cho R3F
Performance — đặc biệt quan trọng cho mobile
  • Instanced Mesh — render 1000 object cùng geometry bằng 1 draw call
  • Level of Detail (LOD) — model ít poly hơn khi ở xa camera
  • Texture compression: KTX2 + Basis — giảm memory GPU
  • DRACO compression — giảm geometry data khi truyền qua mạng
  • Dispose: geometry.dispose(), material.dispose(), texture.dispose()
  • renderer.info — monitor draw calls, triangles, textures
  • Detect GPU tier (detect-gpu) — serve low/high quality scene
Tầng này là đích đến của Awwwards. Bắt đầu với GLSL từ thebookofshaders.com và Bruno Simon's Three.js Journey — đây là 2 resource tốt nhất hiện tại.

SEO & Accessibility — điểm yếu của Web 3D
  • Canvas không indexable — cần hidden semantic HTML song song
  • Dùng aria-label trên canvas element
  • Cung cấp text description cho nội dung 3D quan trọng
  • prefers-reduced-motion: none — tắt animation cho người dùng cần
  • Progressive enhancement: site vẫn hoạt động khi WebGL disabled
  • JSON-LD schema markup ngoài canvas cho product pages
Product Configurator — use case ROI cao nhất
  • 360° product viewer: OrbitControls + limited polar angle
  • Material swap: thay MeshStandardMaterial color/map realtime
  • Variant selection: show/hide mesh groups theo option
  • AR view: WebXR API hoặc model-viewer (Google) — "xem trong phòng"
  • ROI thực tế: conversion rate tăng 30–40% với 3D viewer so với ảnh tĩnh
Virtual Tour — BDS & Architecture
  • Load model căn hộ .glb từ Blender/SketchUp/Revit
  • First-person navigation: PointerLockControls hoặc click-to-move
  • Hotspot system: Raycasting + popup info khi click vào điểm
  • 360° panorama: CubeTextureLoader + skybox
  • Measurement tool: tính khoảng cách giữa 2 điểm trong scene
Data Visualization 3D — ngách học thuật & kỹ thuật
  • Molecular visualization: load PDB file, render atoms + bonds
  • Medical: DICOM slices → volume rendering (Three.js + WebGL3D texture)
  • Network graph 3D: force-directed graph trong không gian
  • Terrain từ heightmap: PlaneGeometry + displacement map
  • Thư viện chuyên biệt: 3Dmol.js (molecules), CesiumJS (globe)
Khi pitch dự án 3D, luôn chuẩn bị số liệu ROI: dwell time, conversion rate, bounce rate so với web thông thường. Chi phí cao hơn 3–5x nhưng kết quả đo lường được.
